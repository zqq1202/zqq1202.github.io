GitHub+HEXO搭建个人博客

参考：

  1. 使用Hexo框架自由搭建博客
  2. 如何用github+hexo搭建个人博客
  3. Hexo官方文档-部署

一、环境

安装hexo之前需要先在windows环境下安装完Node.js和Git。

  1. Windows 8.1 64bit
  2. Node.js 9.3.0
  3. Git 2.15.1.2
  4. GitHub帐号

安装

  1. Node.js的安装
    Node.js安装文件（node-v9.3.0-x64.msi）一路Next。
  2. Git安装
    Git安装文件（Git-2.15.1.2-64-bit.exe）也是一路Next。
  3. 关联Git和GitHub帐号
    已经有Github帐号跳过此步，首先进入Github进行注册，用户名、邮箱和密码之后都需要用到，自己记好。
  4. 创建Repository
    Repository相当于一个仓库，用来放置你的代码文件。首先，登陆进入Github，并进入个人页面，选择Repositories(下图第一个红圈的地方)，然后New一个Repository(下图第二个红圈的地方)

    创建时，只需要填写Repository name即可，当然这个名字的格式必须为yourname.github.io，例如我的为zqq1202.github.io，注意：此处的yourname必须为你的github用户名，用户名是可以用来登陆的。
  5. 添加SSH公钥到github账户
    1） 建一个文件夹(如D:\hexo)
    2） 设置你的用户名和用户邮箱(你github的用户名和你注册github时填的邮箱)
    进入到你所建的文件夹中，右键空白处，选择Git Bash Here
    输入：git config --global user.email "你注册github的邮箱"，然后回车
    再输入：git config --global user.name "你github的用户名"，然后回车
    3） 生成密钥
    继续在上面的窗口输入：ssh-keygen -t rsa -C "你注册github的邮箱"，然后一直回车，直到出现类似下图

    再输入：ssh-agent -s，回车
    输入 ：eval `ssh-agent` -s，回车
    输入：ssh-add，回车
    命令成功执行，则会在默认C盘的用户(users)中的当前用户的.ssh文件夹（C:\Users\qqz\.ssh）中生成两个文件id_rsa.pub和id_rsa。将这个.ssh文件夹拷贝到你Git的安装目录下（C:\Program Files\Git）。
    再回到Git Bash那个窗口，输入：clip < ~/.ssh/id_rsa.pub，回车(这条命令执行后就已经拷贝好你的ssh key了，可以直接粘贴)
    ４) 配置SSH key
    进入GitHub，settings–>SSH and GPG keys–>New SSH key


    在title那里你可以给这个SSH key起个名字，在下面key那里粘贴你刚才拷贝的SSH key
    点击Add SSH key，再输入你github帐号的密码就可以了。
    5） 验证
    接着，检验一下：
    在那个git bash窗口输入：ssh -T git@github.com，回车，出现”are you sure you want to continue connecting?”时输入yes，回车
    如果出现hi,(你github的用户名),you are successfully ·······就证明这个步骤完成。
  6. 安装HEXO
    在桌面右键空白处，选择Git Bash Here
    输入：npm install -g hexo-cli，回车
    检测是否安装成功，在终端输入hexo -v，成功则显示hexo信息

二、搭建博客

  1. hexo初始化
    再到你之前建的文件夹，右键空白处，选择git bash here
    输入：hexo init，回车
    再输入：npm install
  2. 修改hexo配置
    现在我们需要编辑本地D:\hexo文件夹下的 _config.yml 文件，与自己 GitHub 账号的 Repository 仓库建立关联
    再到Git bash那个窗口，输入：vim _config.yml，回车
    找到deploy处，按如下修改（详细配置参考hexo官方文档）:
    title: xxxx //网站标题
    author: xxx //作者名字　　language: zh-Hans //网站使用的语言 中文
    deploy:
    　　type: git
    　　repo: https://github.com/username/username.github.io.git
    　　branch: master
    修改后，按esc退出编辑，输入:wq保存退出。
    基本配置：修改这几个必须设置的键值对，记得保存，注意配置的键值对之间一定要有空格
    键repo： 对应的值就是你 Github 创建的仓库所对应的URL链接。
  3. 生成静态页面
    在你建的文件夹git bash的那个窗口继续输入：hexo generate，回车
    这时可能会弹出一个github登录界面，要你输入你github的用户邮箱和密码，输入即可
  4. 本地测试
    继续在你建的文件夹git bash的那个窗口继续输入：hexo server，回车，本地的服务器启动
    这时，你在浏览器地址栏输入http://localhost:4000就能看到这个blog的初始页面效果啦，但这只是在本地才看得到。
  5. 自动部署发布
    配置文件修改完成以后，执行命令，完成自动部署:
    npm install hexo-deployer-git --save
  6. 同步到github
    在你建的文件夹git bash的那个窗口继续输入：hexo deploy，回车，如果出现 Deploy done:git，则同步成功。
  7. 访问搭建好的博客
    这时你在浏览器地址栏输入你github仓库名，就能看到你的blog啦，搭建成功！

三、新建并发布文章

  1. 新建文章
    如果你此时需要写一篇新的文章，标题是：Hello world，执行命令hexo new "Hello world"就可以了。在D:\hexo\source\_posts 文件夹下面就会生成一个 Hello world.md 文件 ，在 windows 电脑上可以使用 Markdown编辑器打开文件，随意书写。
  2. 发布到GitHub上
    启动本地服务，测试没问题后，我们就可以生成静态网页文件发布至我们的 Github Pages 上。
    hexo clean && hexo g && hexo d
    如果这是你第一次执行，终端会让你输入GitHub 的邮箱和密码，正确输入后，稍等片刻，就会把你的博客上传至GitHub上，个人博客就这样完成了。以后，每次把博客写完后，执行一下这个命令就可以直接发布了。

四、Win下的MarkDown编辑器

  • 作业部落
  • Md2All
  • typora
  • CSDN markdown编辑器
    作业部落可以同步，但是上传图片收费，有客户端和web端，可以实时同步。typora有时会连接不上，有客户端。
    Md2All是和CSDN都是web端。
    上述几个MarkDown编辑器均有MarkDown语法帮助。

五、主题替换

  1. 从GitHub上获取代码
    git clone https://github.com/iTimeTraveler/hexo-theme-hiker.git themes/hiker
  2. 启用
    把Hexo主目录下的 _config.yml 文件中的字段 theme 修改为 hiker。
  3. 配置文件修改完成以后，执行命令，完成自动部署:
    npm install hexo-deployer-git --save
  4. 重新发布页面就可以看到主题了
    hexo clean && hexo g && hexo d
